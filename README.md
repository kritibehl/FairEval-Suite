# FairEval

**Stopped 16 unsafe model releases before deployment.**

Gating GenAI model deployments using regression detection, hallucination scoring, and statistical significance testing.

`Python` · `FastAPI` · `SciPy` · `HuggingFace` · `SQLite`

🔗 **[Live Demo](https://huggingface.co/spaces/kriti0608/FairEval-Suite)** · 📄 **[Technical Report](https://doi.org/10.5281/zenodo.17625268)**

---

## Why This Exists

**Problem:** Average score hides regressions. A model can hold a 0.79 average while silently failing the case class that matters most. Most teams catch quality regressions after deployment.

**Impact:** Unsafe model releases reach users — wrong answers, hallucinations, citation failures, latency regressions. Quality-only gates miss hardware serving failures entirely.

**Proof:** Case-level comparison with Welch t-test and chi-squared significance testing. Reduced false allows from 16 to 0. Caught a +47.1% p95 latency regression on AMD MI300X that passed every quality evaluation.

## Why FairEval

Average score hides regressions. A model can hold a 0.79 average while silently failing the case class that matters most.

Most teams catch quality regressions after deployment. FairEval catches them at the gate:

```
Case-level comparison (not aggregate)
  → Welch t-test (quality delta significance)
  → Chi-squared (pass rate significance)
  → Hardware serving gate (p95 latency)
  → Groundedness validation (citation coverage)
  → Toxicity monitoring

Any signal outside threshold → BLOCK
```

## Architecture

```
Dataset (cases.jsonl)
      │
      ▼
Model adapter  →  Gemini / OpenAI / Anthropic / mock
      │
      ▼
Scorer  →  keyword fallback / RAG-overlap / classification
      │
      ▼
Comparison engine
  avg_score_delta · pass_rate_delta · per-case regressions
  Welch t-test + chi-squared significance
      │
      ▼
Release gate  →  SHIP / BLOCK
  rollback recommendation · affected_query_pct
      │
      ▼
RAI monitoring  →  toxicity · hallucination rate · quality score
```
